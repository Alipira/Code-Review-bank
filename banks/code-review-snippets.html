Companion to the main interview kit. This expands Section 2 (Code Review & Debugging) into a standalone bank of 142 buggy snippets across 16 areas. Each entry has a difficulty tag, the buggy code, and an answer key (Bug + Fix).

How to run one. Paste the snippet, screen-share style: "A junior wrote this. It runs. Find the correctness bugs and performance problems, then fix the hot path." Score the hunt, not the checklist: do they systematically check for leakage, edge cases, dtype, complexity, and train/serve parity — or spot one thing and stop? The best candidates ask why the code is shaped this way before rewriting.

A — NumPy, Vectorization & Numerical Stability (1–10)

1.In-place normalize mutates caller & breaks int arraysE

def normalize(x):
    x -= x.mean()
    x /= x.std()
    return x
Answer key
Bug:

-= / /= modify the caller's array in place (aliasing side effect); on an int dtype, /= truncates or raises.

Fix:

return (x - x.mean()) / x.std(); ensure float input.

2.Broadcasting the wrong axisM

row_centered = X - X.mean(axis=1)   # X is (n, d); mean is (n,)
Answer key
Bug:

(n,d) - (n,) broadcasts against the last axis (or errors) — you did not subtract each row's mean.

Fix:

X - X.mean(axis=1, keepdims=True) → shape (n, 1).

3.Float equality maskE

mask = (predictions == 0.1)
Answer key
Bug:

0.1 isn't exactly representable; equality silently misses matches.

Fix:

np.isclose(predictions, 0.1).

4.Integer overflow in a sum of squaresM

a = np.array([100_000, 200_000], dtype=np.int32)
energy = (a * a).sum()          # overflows int32
Answer key
Bug:

a*a overflows int32 and wraps to garbage silently.

Fix:

cast up: a.astype(np.int64) or float64.

5.Reseeding the global RNG every callM

def sample():
    np.random.seed(42)
    return np.random.randn(10)
Answer key
Bug:

reseeding to a constant returns identical "random" draws each call and mutates global RNG state for everyone else.

Fix:

build one rng = np.random.default_rng(seed) and reuse; never reseed per call.

6.Mean of ratios instead of ratio of meansH

ctr = (clicks / impressions).mean()
Answer key
Bug:

averaging per-row CTR overweights low-impression rows; the true aggregate CTR is clicks.sum()/impressions.sum(). Also divide-by-zero when impressions == 0.

Fix:

pool numerator/denominator; guard zeros.

7.argmax over a flattened arrayE

pred_class = np.argmax(logits)      # logits is (batch, n_classes)
Answer key
Bug:

no axis → argmax over the entire flattened array; returns one scalar.

Fix:

np.argmax(logits, axis=1).

8.Growing an array with np.append in a loopM

result = np.array([])
for row in data:
    result = np.append(result, transform(row))
Answer key
Bug:

np.append reallocates the whole array each iteration → O(n²); plus a Python loop.

Fix:

append to a list then np.array(...), or vectorize.

9.Silent NaN in an aggregateE

score = np.mean(values)
Answer key
Bug:

a single NaN makes the mean NaN silently; the real question is why there are NaNs.

Fix:

np.nanmean (after understanding the NaNs), or handle explicitly.

10.Explicit matrix inverse for least squaresH

beta = np.linalg.inv(X.T @ X) @ X.T @ y
Answer key
Bug:

forming the inverse is numerically unstable and slow; blows up under collinearity (near-singular XᵀX).

Fix:

np.linalg.solve(X.T @ X, X.T @ y) or lstsq/QR; add ridge for collinearity.

B — Pandas & Data Wrangling (11–22)

11.Chained assignment silently no-opsM

df[df['age'] > 30]['income'] = 0
Answer key
Bug:

chained indexing writes to a temporary copy; the original df is unchanged (SettingWithCopy).

Fix:

df.loc[df['age'] > 30, 'income'] = 0.

12.inplace=True returns NoneE

df = df.dropna(inplace=True)
Answer key
Bug:

inplace=True returns None, so df becomes None.

Fix:

either df.dropna(inplace=True) or df = df.dropna() — not both.

13.iterrows for a vectorizable opM

for i, row in df.iterrows():
    df.at[i, 'total'] = row['a'] + row['b']
Answer key
Bug:

iterrows boxes each row into a Series (slow, dtype-coercing) → orders of magnitude slower.

Fix:

df['total'] = df['a'] + df['b'].

14.Merge fan-out explodes rowsH

result = orders.merge(customers, on='customer_id')
Answer key
Bug:

if customers.customer_id isn't unique, the merge multiplies order rows (row explosion), silently corrupting downstream sums.

Fix:

enforce uniqueness or merge(..., validate='many_to_one').

15.groupby drops NaN keysM

counts = df.groupby('category').size()
Answer key
Bug:

rows with a NaN category are dropped by default → undercount.

Fix:

df.groupby('category', dropna=False).

16.NaN silently upcasts ints to floatM

df['count'] = df['count'].fillna(0)   # column already float64 because of earlier NaN
Answer key
Bug:

an earlier NaN forced the column to float, so ids/counts are now 1.0, breaking joins/exports.

Fix:

use nullable Int64, or cast back to int after filling.

17.Arithmetic aligns on index → NaNsM

a = df1['x'].reset_index(drop=True)
combined = a + df2['y']              # df2['y'] index NOT reset
Answer key
Bug:

pandas aligns on index before adding; mismatched indices produce NaNs.

Fix:

align indices or use .values.

18.apply without axis=1E

df['z'] = df.apply(lambda r: r['a'] * 2)
Answer key
Bug:

default axis=0 applies over columns; r['a'] misbehaves.

Fix:

add axis=1 — or better, vectorize df['a'] * 2.

19.Ambiguous datetime parsingM

df['date'] = pd.to_datetime(df['date'])
Answer key
Bug:

mixed/locale-ambiguous strings (01/02/2024) parse inconsistently and slowly without an explicit format.

Fix:

format=..., set dayfirst, errors='coerce', then check nulls.

20.Feature function mutates the caller's frameM

def add(df):
    df['new'] = df['x'] ** 2
    return df
train_feat = add(train)             # `train` is now mutated too
Answer key
Bug:

in-place column write leaks back into the original DataFrame.

Fix:

df = df.copy() at the top.

21.concat with misaligned columnsM

big = pd.concat([df1, df2])
Answer key
Bug:

extra/misordered columns create NaN blocks; duplicate index values break later .loc.

Fix:

align columns, ignore_index=True, validate schema.

22.Reading a giant CSV all at onceM

df = pd.read_csv('50gb.csv')
Answer key
Bug:

loads everything into memory and infers object dtypes that balloon usage.

Fix:

pass dtype=, usecols=, chunksize=, or use Parquet/Dask.

C — Python Core & Complexity (23–32)

23.Mutable default argumentM

def add_item(item, bucket=[]):
    bucket.append(item)
    return bucket
Answer key
Bug:

the default list is created once and shared across all calls → state leaks between calls.

Fix:

bucket=None, then bucket = [] if bucket is None else bucket.

24.Mutating a list while iterating itM

for x in items:
    if x < 0:
        items.remove(x)
Answer key
Bug:

removing during iteration shifts indices and skips elements.

Fix:

items = [x for x in items if x >= 0].

25.Late-binding closure in a loopH

funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])         # [2, 2, 2], not [0, 1, 2]
Answer key
Bug:

closures capture the variable, not its value; all see the final i.

Fix:

bind per-iteration: lambda i=i: i.

26.is for value comparisonE

if x is 1000:
    ...
Answer key
Bug:

is tests identity; only works for small cached ints by accident, fails for large/most values.

Fix:

if x == 1000:.

27.O(n²) membership via listM

seen = []
for x in data:
    if x not in seen:
        seen.append(x)
Answer key
Bug:

x not in list is O(n) → O(n²) overall.

Fix:

use a set for membership.

28.Naive float summation errorM

total = 0.0
for v in values:                    # millions of small floats
    total += v
Answer key
Bug:

sequential summation accumulates floating-point error.

Fix:

math.fsum(values) or np.sum (pairwise/Kahan).

29.Exponential recursion, no memoizationM

def fib(n):
    return n if n < 2 else fib(n - 1) + fib(n - 2)
Answer key
Bug:

exponential time; deep n also overflows the stack.

Fix:

memoize (functools.lru_cache) or iterate.

30.Shallow copy of nested listsM

grid2 = list(grid)                  # grid is a list of lists
grid2[0][0] = 9                     # also mutates grid
Answer key
Bug:

shallow copy shares the inner lists.

Fix:

copy.deepcopy(grid).

31.Bare except: passM

try:
    risky()
except:
    pass
Answer key
Bug:

swallows everything (including KeyboardInterrupt/SystemExit) and hides real failures.

Fix:

catch specific exceptions, log, re-raise as appropriate.

32.Floor division on a ratioE

accuracy = correct // total
Answer key
Bug:

integer floor division → 0 whenever correct < total.

Fix:

correct / total.

D — Data Leakage & Preprocessing (33–42)

33.Scaling before the splitH

X_scaled = StandardScaler().fit_transform(X)
X_train, X_test = train_test_split(X_scaled)
Answer key
Bug:

the scaler is fit on all rows, leaking test mean/variance into training.

Fix:

split first; fit on train, transform test (ideally inside a Pipeline).

34.Global-mean imputation before the splitH

X = X.fillna(X.mean())
X_train, X_test = train_test_split(X)
Answer key
Bug:

the imputation mean includes test rows → leakage.

Fix:

fit the imputer on train only.

35.Target encoding uses the row's own labelH

df['cat_te'] = df.groupby('cat')['y'].transform('mean')
Answer key
Bug:

each encoded value includes its own target and the full dataset → strong leakage, overfit.

Fix:

out-of-fold / leave-one-out target encoding with smoothing, fit on train only.

36.A feature computed from the targetH

df['revenue_per_unit'] = df['revenue'] / df['units']   # revenue IS the label
Answer key
Bug:

the feature encodes the target directly → perfect train metrics, useless in production.

Fix:

drop any feature derived from the label.

37.SMOTE before cross-validationH

X_res, y_res = SMOTE().fit_resample(X, y)
scores = cross_val_score(model, X_res, y_res)
Answer key
Bug:

synthetic points generated from all data leak neighbors of test rows into training folds → optimistic scores.

Fix:

resample inside each CV fold (imblearn Pipeline).

38.Feature selection on the full datasetH

X_sel = SelectKBest(k=20).fit_transform(X, y)   # then split
Answer key
Bug:

selection uses test rows' target signal → leakage and overfit feature set.

Fix:

select inside a pipeline, train-only per fold.

39.Rolling feature that peeks at the present/futureH

df['roll_mean'] = df['y'].rolling(7).mean()
Answer key
Bug:

includes the current row, is unsorted, and mixes entities; without .shift(1) it leaks the label.

Fix:

sort_values('date').groupby(entity)['y'].rolling(7).mean().shift(1).

40.Same entity in both train and testH

train, test = train_test_split(df)    # many rows per user land on both sides
Answer key
Bug:

group leakage — the model memorizes users, inflating metrics.

Fix:

GroupShuffleSplit/GroupKFold on the entity id.

41.Duplicates split across train/testM

train, test = train_test_split(df)    # exact/near-duplicate rows exist
Answer key
Bug:

identical rows in both splits leak labels.

Fix:

dedup (and check near-dupes) before splitting.

42.One-hot encoding fit on all dataM

df = pd.get_dummies(df)               # then split
Answer key
Bug:

leaks category distribution and hides the unseen-category problem at serve time.

Fix:

fit the encoder on train with handle_unknown='ignore'.

E — scikit-learn & Pipelines (43–52)

43.Fitting the scaler on the test setM

scaler.fit(X_test)
X_test = scaler.transform(X_test)
Answer key
Bug:

transforms are fit on test data → leakage.

Fix:

fit on train, transform test.

44.AUC from hard labelsM

roc_auc_score(y_test, model.predict(X_test))
Answer key
Bug:

predict returns 0/1; AUC needs the ranking (scores/probabilities), so it collapses toward accuracy.

Fix:

roc_auc_score(y_test, model.predict_proba(X_test)[:, 1]).

45.Ignoring class weight on imbalanceM

LogisticRegression().fit(X, y)        # y is ~99:1
Answer key
Bug:

default weighting predicts the majority class.

Fix:

class_weight='balanced', plus threshold tuning / resampling.

46.No random_stateE

RandomForestClassifier().fit(X, y)
Answer key
Bug:

nondeterministic results → irreproducible experiments.

Fix:

set random_state=.

47.GridSearch scoring on accuracy for imbalanceM

GridSearchCV(clf, params, scoring='accuracy')   # imbalanced target
Answer key
Bug:

optimizes a metric that rewards predicting the majority.

Fix:

scoring='average_precision' / 'f1' / 'roc_auc'.

48.LabelEncoder on featuresM

X['cat'] = LabelEncoder().fit_transform(X['cat'])
Answer key
Bug:

LabelEncoder is for targets; it imposes a fake ordinal order on nominal categories and can't handle unseen categories at serve.

Fix:

OrdinalEncoder/OneHotEncoder with handle_unknown.

49.Printing negated MSE as MSEE

score = cross_val_score(reg, X, y, scoring='neg_mean_squared_error').mean()
print(f"MSE: {score}")               # prints a negative number
Answer key
Bug:

sklearn returns negative MSE (higher is better); reported value is negative.

Fix:

report -score.

50.Classification split without stratifyM

train_test_split(X, y)               # imbalanced classes
Answer key
Bug:

class ratios differ across splits; the rare class may be absent from the test set.

Fix:

stratify=y.

51.Forgetting the transform at inferenceH

X_train = scaler.fit_transform(X_train)
model.fit(X_train, y_train)
preds = model.predict(X_new)          # X_new never scaled
Answer key
Bug:

train/serve skew — inference features are on a different scale.

Fix:

bundle scaler + model in a Pipeline so serving replays the same steps.

52.Calling fit on the whole dataset then "evaluating"M

model.fit(X, y)
print(accuracy_score(y, model.predict(X)))
Answer key
Bug:

reports training accuracy — no generalization estimate.

Fix:

evaluate on a held-out set / CV.

F — Cross-Validation & Splitting (53–60)

53.k-fold on time seriesH

cross_val_score(model, X, y, cv=5)    # temporal data
Answer key
Bug:

shuffled folds train on the future to predict the past.

Fix:

TimeSeriesSplit.

54.Selecting hyperparameters on the test setH

for p in params:
    model.set_params(**p).fit(X_train, y_train)
    best = max(best, model.score(X_test, y_test))
Answer key
Bug:

tuning against the test set makes it no longer an unbiased estimate.

Fix:

use a validation set / nested CV.

55.Non-grouped CV on grouped dataH

cross_val_score(model, X, y, cv=KFold(5))   # multiple rows per user
Answer key
Bug:

the same entity appears across folds → leakage.

Fix:

GroupKFold on the entity id.

56.KFold instead of StratifiedKFold on imbalanceM

KFold(n_splits=5, shuffle=True)       # imbalanced classification
Answer key
Bug:

folds get uneven class ratios; rare class may vanish from a fold.

Fix:

StratifiedKFold.

57.Default shuffle=False on ordered dataM

KFold(n_splits=5)                     # data sorted by label
Answer key
Bug:

folds become contiguous blocks (e.g., all class A), wrecking the estimate.

Fix:

shuffle=True (unless a deliberate temporal split).

58.Best-of-many CV score reported as the estimateH

# try 50 model configs, report the best CV score
Answer key
Bug:

the maximum over many trials is optimistically biased (multiple-comparisons / snooping).

Fix:

nested CV, and keep a final untouched test set.

59.Test set too small to conclude anythingM

train_test_split(X, y, test_size=0.02)
Answer key
Bug:

a tiny test set gives a high-variance metric; conclusions are noise.

Fix:

adequate test size or repeated CV.

60.Feature window straddles the CV boundaryH

# time-based split, but a 7-day feature window overlaps the train/test cut
Answer key
Bug:

information bleeds across the split boundary (look-ahead at the seam).

Fix:

purge/embargo around the boundary (purged CV).

G — Evaluation & Metrics (61–70)

61.Multiclass F1 with the wrong averageM

f1_score(y_true, y_pred)              # multiclass, default average='binary'
Answer key
Bug:

errors or answers the wrong question; macro/micro/weighted each mean something different.

Fix:

choose average= deliberately (macro for minority-sensitive).

62.Reporting accuracy on imbalance as successE

print("Accuracy:", accuracy_score(y, preds))   # 99%, predicts all-majority
Answer key
Bug:

accuracy is meaningless when the base rate is extreme.

Fix:

PR-AUC, recall, confusion matrix at the operating point.

63.Hardcoded 0.5 thresholdM

preds = (proba > 0.5).astype(int)
Answer key
Bug:

0.5 is arbitrary; wrong for imbalance or asymmetric costs.

Fix:

tune the threshold on validation to the business cost curve.

64.MAPE with zeros in the denominatorM

mape = np.mean(np.abs((y - yhat) / y))
Answer key
Bug:

divide-by-zero when y == 0; explodes for small y; asymmetric penalty.

Fix:

WAPE/sMAPE, or mask zeros.

65.RMSE on a log target, reported in "units"H

model.fit(X, np.log1p(y))
pred = model.predict(X)
rmse = mean_squared_error(np.log1p(y), pred, squared=False)   # not original units
Answer key
Bug:

the error is in log space but interpreted as original units.

Fix:

np.expm1 predictions and targets before computing the metric.

66.Micro-average hides the minority classM

f1_score(y, p, average='micro')       # imbalanced multiclass
Answer key
Bug:

micro is dominated by the majority; minority failure is invisible.

Fix:

macro / per-class breakdown.

67.Wrong positive labelM

precision_score(y, p, pos_label=0)    # the intended positive was 1
Answer key
Bug:

measures precision for the wrong class.

Fix:

set the correct pos_label.

68.Pointwise metric for a ranking taskH

accuracy_score(relevant, predicted_relevant)   # a recommender
Answer key
Bug:

accuracy ignores order/ranking quality.

Fix:

NDCG@k, MAP, recall@k.

69.Comparing models on different folds/seedsM

# model A scored with seed=1, model B with seed=2
Answer key
Bug:

the comparison isn't apples-to-apples.

Fix:

identical folds/seed; paired statistical comparison.

70.Negative R² read as "small but positive"M

r2 = r2_score(y_true, y_pred)         # returns -0.3
Answer key
Bug:

R² can be negative (worse than predicting the mean); a negative value means the model is actively bad, not "weak."

Fix:

interpret the sign correctly; pair with an error metric in units.

H — PyTorch & Deep Learning (71–82)

71.No model.eval() at validationM

def validate(model, loader):
    for x, y in loader:
        out = model(x)                # dropout & BatchNorm still in train mode
Answer key
Bug:

dropout stays on and BatchNorm uses batch stats → wrong validation numbers.

Fix:

model.eval() + wrap in torch.no_grad().

72.Accumulating loss tensors (graph retained → OOM)H

total_loss = 0
for x, y in loader:
    loss = criterion(model(x), y)
    total_loss += loss               # keeps the autograd graph alive
Answer key
Bug:

summing live tensors retains every graph → memory blows up.

Fix:

total_loss += loss.item().

73.nll_loss fed raw logitsM

out = model(x)                        # raw logits
loss = F.nll_loss(out, y)
Answer key
Bug:

nll_loss expects log-probabilities, not logits → wrong loss.

Fix:

F.cross_entropy(out, y) (log_softmax + nll) or apply log_softmax first.

74.Softmax then CrossEntropyLossM

out = torch.softmax(model(x), dim=1)
loss = nn.CrossEntropyLoss()(out, y)
Answer key
Bug:

CrossEntropyLoss applies log-softmax internally; feeding probabilities double-applies it and degrades training.

Fix:

pass raw logits.

75.Tensors on the wrong deviceE

model.cuda()
for x, y in loader:
    out = model(x)                    # x still on CPU
Answer key
Bug:

device mismatch → runtime error (or silent CPU fallback).

Fix:

x = x.to(device); y = y.to(device).

76.BatchNorm with tiny/size-1 batchesH

model.train()
# batch_size = 1 through a BatchNorm network
Answer key
Bug:

BN can't estimate variance from one sample; and using train() at inference uses unstable batch stats.

Fix:

larger batches, eval() at inference, or LayerNorm/GroupNorm.

77.RNN hidden state not detached (unbounded BPTT)H

for x_t in seq:
    h = rnn(x_t, h)                   # graph grows across the whole sequence
Answer key
Bug:

the graph accumulates over all timesteps → OOM and vanishing/exploding gradients.

Fix:

truncated BPTT: h = h.detach() between chunks.

78.Shuffling val, not shuffling trainM

train_loader = DataLoader(train, shuffle=False)
val_loader   = DataLoader(val,   shuffle=True)
Answer key
Bug:

unshuffled training introduces order bias; shuffling validation is pointless.

Fix:

shuffle train, keep val ordered.

79.Divergent learning rate → NaN lossE

optimizer = torch.optim.SGD(model.parameters(), lr=10.0)
Answer key
Bug:

an absurd LR makes the loss diverge to NaN.

Fix:

sane LR, warmup, and gradient clipping.

80.Optimizer bound to a stale modelH

optimizer = torch.optim.Adam(model.parameters())
model = MyNet()                       # re-created AFTER the optimizer
Answer key
Bug:

the optimizer still references the old parameters; the new model never updates.

Fix:

build the optimizer after the final model, or re-bind its param groups.

81.Loss counts padding tokensH

loss = criterion(logits.view(-1, V), targets.view(-1))   # includes PAD positions
Answer key
Bug:

padding positions contribute to the loss → skewed gradients on variable-length sequences.

Fix:

criterion = nn.CrossEntropyLoss(ignore_index=PAD_ID) or mask.

82.No seeding / DataLoader workers reseedM

# no torch.manual_seed, no numpy/random seed, no worker_init_fn
Answer key
Bug:

irreproducible runs; multi-worker DataLoaders reseed independently.

Fix:

seed torch/numpy/random, set worker_init_fn, and torch.use_deterministic_algorithms(True) (mind the perf trade-off).

I — Statistics & Probability (83–90)

83.Population std for a sampleM

sample_std = np.std(sample)           # ddof=0 by default
Answer key
Bug:

NumPy's default ddof=0 gives the biased (population) std for a sample.

Fix:

np.std(sample, ddof=1).

84.Uncorrected multiple comparisonsH

for feature in features:              # 100 features
    if ttest(feature, target).pvalue < 0.05:
        significant.append(feature)
Answer key
Bug:

at α=0.05 you expect ~5 false positives from 100 tests.

Fix:

Bonferroni or Benjamini–Hochberg (FDR).

85.Treating a p-value as an effect sizeM

if p1 < p2:
    feature1_is_more_important = True
Answer key
Bug:

p-value magnitude ≠ effect size or importance (it's confounded with n).

Fix:

report effect sizes with confidence intervals.

86.Pearson correlation to declare "no relationship"M

corr = np.corrcoef(x, y)[0, 1]        # ≈ 0 for a U-shaped relation
Answer key
Bug:

Pearson only captures linear association; a strong nonmonotonic relationship reads as ~0.

Fix:

Spearman / mutual information / actually plot it.

87.Bootstrap without replacementH

boot = np.random.choice(data, size=len(data), replace=False)
Answer key
Bug:

replace=False just permutes the data → every "resample" is identical → zero variance.

Fix:

replace=True.

88.Peeking at an A/B test and stopping earlyH

if ttest(a, b).pvalue < 0.05:
    stop_and_ship()
Answer key
Bug:

repeated peeking inflates the Type I error rate far above 5%.

Fix:

fixed sample horizon or a sequential/always-valid test.

89.t-test on tiny, skewed samplesM

ttest_ind(a, b)                       # n = 8 each, heavy skew
Answer key
Bug:

the t-test's normality assumption is violated at tiny n with skew.

Fix:

Mann–Whitney U or a bootstrap.

90.Pooling groups hides Simpson's paradoxH

overall_rate = df['success'].mean()   # ignores a confounding group variable
Answer key
Bug:

the aggregate can reverse the within-group trend.

Fix:

stratify by the confounder / adjust.

J — Time Series (91–98)

91.Random split on temporal dataH

train_test_split(ts_df)
Answer key
Bug:

shuffling time lets the model see the future → leakage.

Fix:

chronological split.

92.Standardizing with whole-series statsH

ts['z'] = (ts['y'] - ts['y'].mean()) / ts['y'].std()
Answer key
Bug:

uses future values' mean/std → leakage.

Fix:

expanding/rolling stats, or fit on the train window only.

93."Lag" feature that isn't laggedM

ts['lag_target'] = ts['y']            # named a lag, but not shifted
Answer key
Bug:

uses the current value → direct target leakage.

Fix:

ts['y'].shift(1).

94.Rolling over an unsorted, multi-series frameH

ts['roll'] = ts['y'].rolling(7).mean()   # rows unsorted, several series stacked
Answer key
Bug:

unordered windows + contamination across different series.

Fix:

sort by time, groupby(series_id) before rolling, then shift.

95.Forecasting on differences, forgetting to invertH

model.fit(diff_series)
forecast = model.predict(...)         # reported as a level
Answer key
Bug:

the forecast is in differenced space; reporting it as a level is wrong.

Fix:

cumulatively invert the differencing.

96.Backfilling leaks the futureM

ts['y'] = ts['y'].fillna(method='bfill')
Answer key
Bug:

bfill fills past gaps with future values → leakage.

Fix:

ffill (or model-based imputation), and handle edge rows.

97.Resample window overlaps the labelH

weekly = ts.resample('W').mean()      # features and target share the same week
Answer key
Bug:

the aggregation window overlaps the prediction target → leakage.

Fix:

define feature and label windows with a gap between them.

98.CV horizon shorter than the seasonal cycleM

TimeSeriesSplit(n_splits=5)           # tiny test horizon vs yearly seasonality
Answer key
Bug:

a test horizon shorter than the seasonal period can't capture seasonality → misleading CV.

Fix:

horizon ≥ one seasonal cycle; blocked/seasonal CV.

K — NLP / Embeddings / LLM / RAG (99–108)

99.TF-IDF fit on the full corpusH

X = TfidfVectorizer().fit_transform(all_texts)   # then split
Answer key
Bug:

vocabulary and IDF weights are learned from test documents → leakage.

Fix:

fit the vectorizer on train only.

100.Dot product mistaken for cosine similarityM

sim = a @ b                           # a, b are unnormalized embeddings
Answer key
Bug:

dot product ≠ cosine unless the vectors are unit-normalized; magnitude dominates ranking.

Fix:

normalize first, or use a cosine function.

101.Silent truncation dropping contextH

inputs = tokenizer(text, truncation=True, max_length=128)   # long docs
Answer key
Bug:

long documents are silently cut, losing the content you need for retrieval/classification.

Fix:

chunk the document (or raise max length) and log when truncation happens.

102.Query and documents embedded by different modelsH

doc_emb   = model_A.encode(docs)
query_emb = model_B.encode(query)     # different embedding space
Answer key
Bug:

query and corpus live in different vector spaces → retrieval is garbage.

Fix:

use the identical model (and version) for both.

103.Fixed-character chunking with no overlapM

chunks = [text[i:i+500] for i in range(0, len(text), 500)]
Answer key
Bug:

character windows split words/sentences and have no overlap → broken, context-less retrieval units.

Fix:

token-aware chunking on sentence/semantic boundaries with overlap.

104.Context assembled without dedup or a token budgetH

context = "\n".join(retrieved)        # may exceed the context window
Answer key
Bug:

duplicate chunks, no reranking, and silent truncation when the prompt overflows the window.

Fix:

rerank, dedup, and enforce an explicit token budget.

105.Evaluating generation with exact matchM

score = (generated == reference)
Answer key
Bug:

string equality is near-useless for free-form generation.

Fix:

semantic/faithfulness metrics, groundedness (NLI vs. context), LLM-as-judge, periodic human eval.

106.High temperature for a factual/extraction taskE

llm(prompt, temperature=1.2)          # structured extraction
Answer key
Bug:

high temperature makes structured/factual outputs unstable and hallucination-prone.

Fix:

temperature ≈ 0 for deterministic tasks.

107.Untrusted content concatenated into the promptH

prompt = system_instructions + user_uploaded_doc + question
Answer key
Bug:

retrieved/user text can override your instructions (prompt injection).

Fix:

separate roles, delimit untrusted content, add instruction-defense and output filtering, restrict tool access.

108.Stale index after an embedding-model upgradeH

# upgraded the embedding model but did NOT re-embed the corpus
Answer key
Bug:

new query vectors are compared against an index built by the old model → retrieval quality collapses.

Fix:

re-embed the whole corpus on any model change; version the index alongside the model.

L — Recommender Systems (109–114)

109.Random split for a temporal recommenderH

train_test_split(interactions)
Answer key
Bug:

future interactions predict past ones → leakage; offline metrics won't hold online.

Fix:

time-based split (e.g., leave-last-interaction-out).

110.Crediting already-seen items as hitsM

# recommend items the user already interacted with in train, count them as correct
Answer key
Bug:

recommending known items inflates hit-rate/recall.

Fix:

exclude previously-seen items from candidates and evaluation.

111.Cold-start KeyError at serveM

user_vec = embeddings[user_id]        # KeyError for a brand-new user
Answer key
Bug:

no handling for unseen users/items in production.

Fix:

fallback (popularity/content-based) and a reserved UNK embedding.

112.Negative sampling that hits true positivesM

neg = random.choice(all_items)        # might be an item the user actually likes
Answer key
Bug:

sampled "negatives" can be real positives → noisy labels.

Fix:

exclude the user's known positives when sampling.

113.Metric over the full catalog, not top-kM

recall = hits / total_relevant        # across the whole catalog
Answer key
Bug:

not a top-k metric; misrepresents ranking quality.

Fix:

recall@k / NDCG@k / MAP@k.

114.Implicit feedback treated as explicit ratingsH

model.fit(clicks_as_ratings)          # click = 1, but no-click ≠ dislike
Answer key
Bug:

absence of interaction isn't a negative; treating implicit signals as explicit ratings mismodels preference.

Fix:

implicit MF with confidence weighting and proper negative handling.

M — Spark / PySpark & Big Data (115–120)

115..collect() on a huge DataFrameM

data = df.collect()                   # pulls every row to the driver
Answer key
Bug:

driver OOM.

Fix:

aggregate/take(n)/write to storage; keep work distributed.

116.Python UDF where a built-in existsM

@udf
def add_one(x): return x + 1
df = df.withColumn('y', add_one('x'))
Answer key
Bug:

row-by-row Python UDFs serialize to/from the JVM and kill performance.

Fix:

native col('x') + 1, or a vectorized pandas_udf.

117.Repeated actions recompute the lineageM

for i in range(10):
    print(df.filter(cond).count())    # recomputed from scratch each time
Answer key
Bug:

no caching → the whole DAG re-executes every action.

Fix:

.cache()/.persist() when a DataFrame is reused.

118.Skewed groupBy with a hot keyH

df.groupBy('key').agg(...)            # one key holds 90% of rows
Answer key
Bug:

data skew creates a straggler task that dominates runtime.

Fix:

key salting / enable Adaptive Query Execution.

119.toPandas() on the whole distributed frameM

pdf = df.toPandas()                   # entire dataset into driver memory
Answer key
Bug:

OOM on the driver.

Fix:

sample or aggregate first.

120.monotonically_increasing_id used as a stable keyH

df = df.withColumn('id', monotonically_increasing_id())   # later used as a join key
Answer key
Bug:

the ids aren't contiguous and aren't stable across recomputation/partitioning — unsafe as a durable key.

Fix:

hash of business keys, or row_number() over a defined window.

N — MLOps, Serving & Reproducibility (121–128)

121.Divergent preprocessing between train and serveH

# training: df.fillna(df.mean()); serving code omits it entirely
Answer key

features differ at inference (train/serve skew).

Fix:

one shared transform artifact (Pipeline / feature store).

122.No seeds anywhere in the training artifactM

the trained model isn't reproducible.

123.Pickling a model without pinning the environmentM

Answer key

cold-loading on each call adds huge latency.

125.No input validation at the serving boundaryH

def predict(payload):

missing/extra features, wrong order, or NaNs produce garbage or crashes; schema drift goes unnoticed.

126.Logging raw payloads with PIIM

127.Retraining on the model's own predictionsH

Answer key
Bug:

a self-reinforcing feedback loop entrenches the model's own bias (label leakage over time).

128.Deploy-and-forget: no drift monitoringH

# ship the model; no checks on inputs, predictions, or KPI afterward
Answer key

silent degradation as the world shifts.

O — SQL (129–136)

129.COUNT(col) skips NULLsE

SELECT COUNT(commission) FROM sales;   -- expecting a row count
Answer key
Bug:

COUNT(column) ignores NULLs, undercounting rows.

Fix:

COUNT(*) for rows.

130.NOT IN with a NULL in the subqueryH

SELECT * FROM a WHERE id NOT IN (SELECT id FROM b);   -- b.id contains NULL
Answer key
Bug:

a single NULL makes NOT IN return no rows at all.

Fix:

NOT EXISTS, or exclude NULLs from the subquery.

131.Filtering the right table of a LEFT JOIN in WHEREH

SELECT * FROM a LEFT JOIN b ON a.id = b.id WHERE b.status = 'active';
Answer key
Bug:

a WHERE predicate on the right table silently converts the LEFT JOIN into an INNER JOIN (drops unmatched rows).

Fix:

move the condition into the ON clause.

132.Aggregate ignores NULL / integer divisionM

SELECT AVG(score) FROM t;              -- NULLs silently excluded
Answer key
Bug:

AVG drops NULLs (maybe unintended); manual SUM()/COUNT() can also do integer division in some engines.

Fix:

COALESCE to decide NULL handling; cast to avoid integer division.

133.Non-aggregated column not in GROUP BYM

SELECT dept, name, MAX(salary) FROM emp GROUP BY dept;
Answer key
Bug:

name is neither grouped nor aggregated → nondeterministic (MySQL) or an error (standard SQL).

Fix:

use a window function (ROW_NUMBER over salary DESC) to pick the row.

134.Join fan-out double-counts a SUMH

SELECT SUM(o.amount) FROM orders o JOIN items i ON o.id = i.order_id;
Answer key
Bug:

a 1-to-many join repeats o.amount once per item → inflated total.

Fix:

aggregate items first, or sum a de-duplicated order grain.

135.BETWEEN on timestamps misses the last dayM

WHERE ts BETWEEN '2024-01-01' AND '2024-01-31';
Answer key
Bug:

'2024-01-31' becomes midnight, so any time later that day is excluded.

Fix:

ts >= '2024-01-01' AND ts < '2024-02-01'.

136.DISTINCT used to hide a join bugM

SELECT DISTINCT customer_id FROM ... ;   -- papering over a fan-out
Answer key
Bug:

DISTINCT masks real duplication (a bad join) instead of fixing it — and hides genuine data issues.

Fix:

find the root cause of the duplication.

Scoring these challenges

Score the process, then the findings:

SignalWeak (1–2)Strong senior (4–5)
Search patternSpots one bug, declares doneSweeps systematically: correctness → edge cases → dtype/NaN → complexity → train/serve parity

ComplexityIgnores itStates Big-O, identifies the hot path, proposes the right structure

Tiering by round: use E items on a phone screen (can they read code at all?), M items in the core coding round, and cluster H items — especially the leakage (D, F), PyTorch (H), and RAG (K) ones — to separate seniors from mid-levels. Two or three H snippets tell you more than twenty E ones.

Universal green flags: narrates a repeatable checklist; names leakage/skew without prompting; quantifies complexity; writes a minimal fix and a test; distinguishes "bug" from "smell"; asks about inputs and scale before rewriting.

Universal red flags: fixates on style over correctness; "just retrain / just rewrite"; reaches for .fit() on the whole dataset; can't estimate Big-O; misses that the code mutates its inputs; treats a passing run as proof of correctness.
